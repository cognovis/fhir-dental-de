The Problem

Data Silos in Dental Practice

  • Patient data locked in proprietary PVS systems (Dampsoft, CGM/Solutio, EVIDENT, medatixx)
  • No standardized data exchange between dental software
  • SWS 2.0 (system migration format) is flat-file based, not FHIR
  • AI companies (DentalXrai, Overjet) need structured data but get PDFs

No Dental FHIR Standard in Europe

  • KZBV has not published dental FHIR profiles
  • VDDS focuses on legacy GDT/DICOM standards
  • ePA 3.0 (mandatory since Jan 2025) uses FHIR — but no dental specialization
  • HL7 Dental Data Exchange IG exists for US only (CDT codes, US Core)

The Solution: Open FHIR Infrastructure

Two open-source Implementation Guides that provide the missing interoperability layer for German dental and medical practice.

fhir-praxis-de

Foundation layer for all German medical practices

  • Shared extensions (billing quarters, fee schedules)
  • Organization & practitioner profiles
  • Insurance & encounter patterns

fhir-dental-de

Complete dental specialization with 16 profiles

  • Clinical findings (FDI tooth chart, periodontal)
  • Procedures (BEMA/GOZ), billing, imaging, lab
  • Treatment plans (HKP, PAR, KFO, ZE)

World First: Clinical Dental Lab Orders in FHIR

DentalLabServiceRequestDE — the first FHIR profile combining billing and clinical specifications for dental lab orders.
Analogous to VisionPrescription for optometry — but for dental prosthetics.

The Gap (globally)

  • VDDS Laborschnittstelle & eLABZ: billing only (BEL II / beb’97)
  • ISO 18618: CAD/CAM file transfer (technical, not clinical)
  • HL7 Dental Data Exchange (US): no lab order profile
  • Clinical specs (shade, material, preparation): paper slips & free text

What We Model

  • Restoration type — 23 codes (Krone, Brücke, Teleskop, Schiene…)
  • Tooth shade — VITA Classical A1-D4 (20 shades incl. Bleach)
  • Material — 15 codes (Zirkon, NEM, Lithiumdisilikat, PEEK…)
  • Preparation form — Stufe, Hohlkehle, Tangential…
  • Occlusion concept — Canine guidance, group function…
  • Implant system — Manufacturer, abutment, platform ø
  • Digital assets — STL scans, photos, bite registration

6 new CodeSystems · 7 clinical Extensions · FDI tooth identification · extensible bindings for international use

International by Design

fhir-dental-de uses a dual-coding strategy: German codes as primary binding, international codes as secondary.

Tooth Identification

Primary: FDI Two-Digit (ISO 3950) — used by 150+ countries

Secondary: SNOMED CT anatomical concepts — bridges to US ADA numbering

Every tooth reference carries both codings, enabling cross-border interpretation.

Clinical Observations

Primary: LOINC + SNOMED CT (extensible binding)

Secondary: German-specific codes only where no international equivalent exists

Periodontal findings, caries assessment, and oral hygiene use universally recognized clinical concepts.

Result: Clinical profiles are 80% DACH-compatible today. Only billing and insurance workflow profiles require national adaptation.

Harmonization Opportunity: Netherlands

Shared Ground

  • Both use SNOMED CT for clinical concepts
  • Both model periodontal screening (DE: granular; NL: PSI aggregate)
  • Both model oral hygiene assessment
  • Both use single unified Procedure profile (not per-specialty)
  • Both are FHIR R4, open-source, actively maintained

Concrete Deliverables

  • European SNOMED CT dental ValueSet — shared terminology for findings, conditions, procedures
  • Aligned periodontal model — PSI screening (NL) + detailed probing (DE) as two conformance levels
  • FDI tooth identification adoption in MedMij (currently not exposed)
  • Joint HL7 ballot proposal for European dental FHIR track

Two independent European IGs arriving at compatible structures = strong signal for HL7 International.

How to Get Involved

PVS Vendors

Adopt fhir-dental-de profiles for data export. Use mira-adapters as reference implementation.

Benefit: Your customers can connect to any FHIR-capable system without you building FHIR.

AI / MedTech Companies

Use fhir-dental-de as the target schema for structured dental data. Contribute profile requirements.

Benefit: One integration works across all PVS systems that adopt the standard.

Standards Organizations

Collaborate on European harmonization. Joint SNOMED CT dental terminology. HL7 ballot sponsorship.

Benefit: Build on working code, not PowerPoint specifications.
