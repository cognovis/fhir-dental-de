# Background: ExamplePatient declares conformance to KBV_PR_FOR_Patient via
# meta.profile. That profile enforces the following constraints that conflict
# with IG Publisher tooling behaviour:
#
# 1. Patient.text (max=0): KBV_PR_FOR_Patient forbids narrative text on

#    resulting "max allowed is 0 but found 1" validation error.
#
# 2. Patient.identifier / Patient.name slice-matching: KBV_PR_FOR_Patient
#    uses closed slicing on identifier and name elements. The basic KVID-10
#    identifier structure and standard HumanName used in ExamplePatient match
#    more than one discriminator slice, producing expected "matches more than
#    one slice" warnings that do not indicate a real data error.
#
# These suppressions apply broadly to all Patient instances validated in this
# IG because the IG Publisher does not support per-instance suppression.
# They are intentional and should only be removed if KBV_PR_FOR_Patient
# conformance is dropped from ExamplePatient or if the upstream constraints
# change.
# ---------------------------------------------------------------------------
